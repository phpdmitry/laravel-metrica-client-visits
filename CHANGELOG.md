# Changelog

## 2.0.0

- Публичный API заменён на `VisitImporter`, `VisitImportRequest` и `VisitLookup`.
- Визиты сохраняются в постоянной модели `Visit`; внутренние события доступны через `VisitEvent` с отношениями `visits` и `primaryVisit`.
- Добавлена миграция данных из старых кандидатов и выбранных совпадений.

## v0.2.2 — 2026-07-23

- Добавлена безопасная data-миграция: для старых `VisitMatch` восстанавливает `visit_started_at` из связанного `VisitCandidate`, сохранённого в UTC.

## v0.2.1 — 2026-07-23

- Исправлена работа временных меток событий и визитов при `APP_TIMEZONE`, отличном от UTC.
- UTC-значения в `metrica_visit_events`, `metrica_visit_candidates` и `metrica_visit_matches` теперь гидратируются как UTC независимо от часового пояса приложения.
- Существующие UTC-строки интерпретируются корректно без миграции и без изменения данных.

## v0.2.0 — 2026-07-23

- Сохраняются и выдаются все найденные визиты ClientID через `VisitCandidate`.
- Повторный завершённый или failed запрос с тем же `externalId` создаёт новую выгрузку.
- Добавлена объединённая история кандидатов по `externalId` без дублей `visit_id`.
- Добавлена необязательная стратегия выбора `first`; прежняя стратегия `last` остаётся значением по умолчанию.
